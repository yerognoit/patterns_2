# О программе:

Данное приложение на примере магазина одежды демонстрирует преимущества использования паттернов "Абстрактная фабрика" и "Хранитель" для создания гибкой, расширяемой и поддерживаемой архитектуры кода. Эти паттерны помогают управлять сложностью и улучшить модульность приложения.
Оба паттерна способствуют улучшению модульности, гибкости и расширяемости кода, что делает его более поддерживаемым и масштабируемым.

# Абстрактная фабрика
Цель паттерна: предоставить интерфейс для создания семейств взаимосвязанных или взаимозависимых объектов без указания их конкретных классов.
В коде: IClothingFactory, MenClothingFactory, WomenClothingFactory представляют абстрактную фабрику и ее конкретные реализации для создания различных типов одежды.
Преимущества: облегчает замену семейства продуктов, гарантирует совместимость создаваемых объектов, изолирует клиентский код от деталей создания объектов.
Паттерн "Абстрактная фабрика" используется для создания семейства объектов (рубашек и брюк) в зависимости от выбранного стиля (мужской или женский). Это позволяет легко добавлять новые типы одежды или варианты стилей, не изменяя клиентский код.

# Хранитель
Цель паттерна: позволяет сохранять и восстанавливать предыдущее состояние объекта без нарушения его инкапсуляции.
В коде: ClothingMemento представляет хранителя, который сохраняет список одежды в определенный момент времени. ClothingCaretaker является опекуном, который управляет сохранением и восстановлением состояния.
Преимущества: обеспечивает сохранение и восстановление состояния объекта без нарушения инкапсуляции, упрощает откат операций, позволяет клиентскому коду работать с объектом, не зная о его деталях состояния.
Паттерн "Хранитель" используется для сохранения состояния списка одежды и восстановления его в будущем. Это полезно, например, для отката изменений или сохранения состояния перед выполнением определенных операций.

# Паттерны:

Данная программа использует такие паттерны как: паттерн абстрактной фабрики и хранителя.
